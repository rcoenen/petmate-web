# Character Matching & Selection Techniques

Research notes on algorithms for matching source image cells to fixed character sets.

---

## Current Approach: Brute-Force with Early Exit

For each 8x8 cell, test all 256 characters x 15 fg colors. Score by weighted Lab error + luminance penalty + repeat penalty. Early-exit when background error alone exceeds current best.

This is correct and high quality. The research below covers acceleration and refinement techniques.

---

## Acceleration: Reducing the Search Space

### Fill-Density Pre-Clustering

Characters can be grouped by their set-pixel count (already computed as `refSetCount`). For a given cell, estimate the target fill density from source luminance vs. bg/fg luminance, then search only the 2-3 nearest density bins.

With 256 characters binned into ~10 density groups, this cuts the search by ~3-4x. Low risk — the luminance penalty already biases toward correct density, so candidates far from the target density would lose anyway.

**Implementation**: Sort characters by `refSetCount`, create bin boundaries. For each cell, compute ideal density from `(srcAvgL - bgL) / (fgL - bgL)`, find the matching bin, search that bin +/- 1.

### Decision Tree Classifier (Markus et al. 2015)

"Fast Rendering of Image Mosaics and ASCII Art" (Computer Graphics Forum, 2015) trains a binary decision tree offline. Each node does a single pixel-intensity comparison ("is pixel[x,y] > threshold?"), each leaf maps to a codebook tile.

Training data is generated by brute-force converting many images using SSIM as quality metric. At runtime, tile selection is O(log N) pixel comparisons — enables real-time video. The PixLab ASCII art library (https://art.pixlab.io/) implements this with WASM support.

**Trade-off**: Loses the fg color co-optimization. Good for speed, not for maximum quality. More suited to a "fast preview" mode than final output.

### Feature-Vector Clustering

Encode each character as a feature vector:
- **Raw 64-bit bitmap**: Hamming distance in a VP-tree
- **4x4 downsampled grayscale**: KD-tree over 16D vectors
- **HoG (Histogram of Oriented Gradients)**: 8-bin orientation histogram, k-NN

A 2025 paper ("Evaluating ML Approaches for ASCII Art") found k-NN with HoG features matches CNN quality. But for our fixed 256-character set, brute-force with early exit is already fast enough that the KD-tree overhead may not help.

### Vector Quantization with Tree-Structured Codebook

Build a binary tree over the 256 characters: at each node, split by projection onto max-variance axis. Query by tree traversal — O(log 256) = O(8) comparisons. Classical TSVQ literature shows this works well for codebooks of this size. Again, more useful for real-time than batch.

## Dithering at the Character-Cell Level

### Block Error Diffusion

From the libcaca study (http://caca.zoy.org/study/part3.html):

After matching each cell, compute the aggregate Lab error (difference between source cell average and rendered cell average). Distribute that error to neighboring cells using Floyd-Steinberg weights (7/16 right, 3/16 below-left, 5/16 below, 1/16 below-right). Neighboring cells' source pixel values are biased before their own matching step.

**Sub-block diffusion** (same source) constrains the kernel to avoid overshooting, producing smoother results than full FS propagation.

This is the most principled approach. It introduces a serial dependency (cells must be processed in scanline order), but that's already the case in the current converter.

### Pre-Dither-then-Match

Apply ordered dithering (Bayer 4x4 or 8x8) to the scaled source image as preprocessing, before cell-level matching. This transforms pixel values before quantization, introducing spatial variation that survives through tile matching.

RetroPixels (https://github.com/micheldebree/retropixels) defaults to Bayer 4x4 and finds it generally produces the best results. Simpler than error diffusion and has no serial dependency.

### Chafa's Grain-Based Dithering

Chafa supports dithering with configurable "grain" dimensions (1x1 through 8x8). An 8x8 grain modulates entire character cells. The threshold matrix adds a spatially-varying bias to source luminance before matching, causing adjacent cells to select different characters and creating a dithering effect across the screen.

### Yliluoma's Arbitrary-Palette Positional Dithering

https://bisqwit.iki.fi/story/howto/dither/jy/

Pre-computes candidate color mixtures for each source color. For each pixel, a set of N candidate palette colors is chosen such that their weighted average approximates the source. A Bayer threshold matrix selects which candidate to use at each position.

Adaptable to cell level: for each cell, compute the ideal fg/bg blend, find candidate character+color combos at different fill levels, use an 8x8 threshold matrix to select among them.

## Multi-Pass Refinement

### Iterative Background Refinement (Dithertron)

Dithertron (https://github.com/sehugg/dithertron) iterates: pick initial cell colors, dither, re-evaluate cell colors, repeat until stable. This is coordinate-descent optimization — fix dithered pixel values, re-choose block colors, re-dither, repeat.

### Alternating Optimization for MCM

Fix global colors, optimize per-cell assignments. Then fix assignments, try swapping each global color. Repeat 2-4 iterations. Each step is guaranteed non-increasing in total error. This is the technique Dithertron uses ("iterate until they stabilize").

### High-Error Cell Revisiting

After the initial greedy pass, identify the top 10-20% of cells by error. Re-evaluate with a wider search (all 16 fg colors without early-exit, or try inverting fg/bg). Catches cases where early-exit pruning was too aggressive.

### Genetic Algorithm (Syntiac)

https://www.syntiac.com/tech_ga_c64.html

GA over cell assignments. Mutations flip individual cells, crossover swaps rectangular regions. Fitness = total image error. Can escape local minima but slow (many generations). Research (Scheunders 1997) showed GA beats k-means for small palette sizes.

## Saliency: Alternatives to Deviation-from-Mean

### Current: Lab deviation from cell mean

```
weight[p] = 1.0 + alpha * (deviation[p] / max_deviation)
```

Good general measure. Weakness: conflates noise with edges. A noisy flat area gets high weights on noise pixels.

### Gradient Magnitude (Sobel/Scharr)

Compute gradient magnitude at each pixel. Edge pixels (high gradient) get higher weight. More precisely targets edges — transitions between regions — which is what the eye tracks. Doesn't amplify noise in flat areas.

### SSIM-Based Matching

Replace or supplement per-pixel Lab error with SSIM (Structural Similarity Index). SSIM compares luminance, contrast, and structure between patches. Evaluates whether the rendered cell preserves the *pattern* of light and dark, not just per-pixel color. Markus et al. (2015) found SSIM produces better perceptual results than MSE for character art.

### Frequency-Based (DCT)

Compute DCT of each 8x8 cell. High-frequency coefficients = texture/detail, low-frequency = flat. Weight errors by frequency magnitude. Cells with fine texture might benefit from different matching strategies than gradient cells.
